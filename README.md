### JDK 와 JRE

 JDK :자바 프로그램을 개발하기 위한 도구 모음

 JRE : 자바 프로그램을 실행하기 위한 환경

 
- JDK의 Bin 디렉터리에 포함된 주요 개발 도구 와 기능 설명

  javac : 자바 소스( .java )를 자바 컴파일러( javac )를 통해 바이트 코드( .class )로 컴파일한다.
  
  [ 사람의 언어( .java )를 컴퓨터의 언어(.class)로 통역(javac)하는 것이다. ]

  java : 자바 응용프로그램 실행기. 자바 JVM 를 작동시켜 자바프로그램으로 실행

  javadoc : 자바 소스로부터 HTML 형식의 API 도큐먼트를 생성한다.

  [ 소스 코드 주석으로 HTML 문서 제작 하는 용도 ]

  jar : 자바 클래스들(패키지포함)을 압축한 저버 아카이브 파일(.jar)을 생성 관리한다.

  [ 클래스 파일들을 하나로 압축시켜준다 ( .jar ) ]
  
  jmod : 자바의 모듈 파일(.jmod)을 만들거나 모듈 파일의 내용 출력을 한다.

  jlink : 응용프로그램에 맞춘 맞춤형(custom) JRE 제공한다.

  jdb : 자바 응용프로그램의 실행 중 오류를 찾는 데 사용하는 디비거이다.

  [ 말한대로 프로그램 오류를 찾는 디버깅 도구이다 ]

  javap : 클래스 파일의 바이트 코드를 소스와 함께 보여주는 디어셈블러이다.
  
  ### java 표준안

- Java SE ( Standard Editon )
  
  데스크톱, 서버, 임베디드 시스템을 위한 표준 자바 플랫폼

  java.lang.*, java.util.*, java.awt.*, javax.rmi.*, javax.net.* 등 대부분의 패키지가 포함되어 있는 대중적 자바 플랫폼이다.


- Java EE(Enterprise Edition)

    자바를 이용한 서버측 개발을 위한 플랫폼

    JSP, Servlet, JDBC, JNDI, JTA, EJB 등 웹 프로그래밍에 필요한 기능을 포함하고 있는 플랫폼으로 Java SE 플랫폼을 기반으로 그 위에 탑재되어있다.


 - Java ME(Micro Edition)

   임베디드를 위한 자바 플랫폼이며 휴대전화, PDA, 셋톱박스 등에서 Java프로그래밍 언어를 지원하기 위해 만들어진 플랫폼으로 Java SE 플랫폼을 기반으로 그 위에 탑재되어있다.

### Java의 네임 방식

- 초기 버전 표기법은 1.x ( 1.0,1.1,1.2 등등... ) 방식으로 표기되어있다.

- 1.2 버전에서 J2SE( Java Standard Edition 6 ) 형태로 변경되었다,

- 1.6 버전부터 Java SE 6 ( Java Standard Edition 6 ) 형태로 변경 되었습니다.

-내부 버전 표기는 1.x.x 형태로 유지되고 있다.


Java 9 부터 시작된 모듈 프로그래밍

- 모듈화 : Java.9에서 정의된 새로운 기능이며 2017년 9월 21일에 출시하였다.

- 모듈 프로그래밍 : 필요한 모듈을 연결하는 방식으로 작성

- 자바 플랫폼의 모듈화

  - 실앻 시간에 사용되는 자바 API의 모든 클래스들을 모듈로 분할

  - 세밀한 모듈화, 자바 응용프로그램이 실행되는데 필요 없는 모듈 배제

  - 작은 크기의 실행 환경 구성

### 자바의 특징

- 실행 코드 배포

    - 한 개의 class 파일 또는 다수의 class 파일로 구성

    - 여러 폴더에 걸쳐 다수의 클래스 파일로 구성된 경우 : jar 압축 파일로 배포한다.

    - 자바 응용프로그램의 실행은 main() 메소드에서 시작한다.

    - 하나의 클래스 파일에 두개 이상의 main() 메소드가 있을 수 없다.

    - 각 클래스 파일이 main() 메소드를 포함하는 것은 상관없다.

- 패키지

    - 서로 관련 있는 여러 클래스를 패키지로 묶어 관리

    - 패키지는 폴더 개념이다.

- 멀티 스레드

    - 여러 스레드의 동시 수행 환경 지원

    - 자바는 운영체제의 도움 없이 자체적으로 멀티 스레드를 지원한다.

    - C/C++ 프로그램은 멀티 스레드를 위해 운영체제 API를 호출한다.

- 가비지 컬렉션
    
    - 자바 언어는 메모리 할당 기능은 있어도 메모리 반환 기능은 없다.

    - 사용하지 않는 메모리는 JVM에 의하여 자동 반환한다 - 가비지 컬렉
